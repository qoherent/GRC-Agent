# Inspect Graph Experiment Output Comparison Analysis

This document compares the JSON structure and contents generated by the legacy inspection tool (`run_experiment.py`) with those generated by the native `gnuradio.grc.core` Python API (`verify_native_api.py`) using the strict native filtering rules.

---

## 1. Top-Level Structure Comparison

| Key | Present in Legacy | Present in Native | Description |
|---|---|---|---|
| `tool` | Yes | Yes | Identifies the tool name (`"inspect_graph"`). |
| `ok` | Yes | Yes | Status boolean indicating successful tool execution. |
| `params` | Yes | No | Nested dictionary of block parameters (by block `instance_name`). Native inlines these. |
| `graph` | Yes | No | Contained overview lists of blocks, connections, and validation status. |
| `options` | No | Yes | Contains flowgraph-wide option block parameters (e.g., `output_language`, `generate_options`). |
| `blocks` | No | Yes | A dict of blocks with `block_type`, `role`, and inlined `parameters`. |
| `connections`| No | Yes | List of active connections (top-level). |
| `validation` | No | Yes | Contains validation `status` and full compiler error list (`errors`). |

---

## 2. Block & Connection Presence

### Blocks:
- **Present in Native but Missing in Legacy**: 
  - The `options` block (e.g., `"random_bit_generator"`, `"dial_tone"`, `"mac_sniffer"`, etc.) is returned in native with the role `"metadata"`. In legacy, the options block was excluded from the block list.
- **Present in Legacy but Missing in Native**:
  - `qtgui_time_sink_x_1` in `random_bit_generator_dual_sink_sink1_disabled.grc`.
  - **Reason**: The block is disabled (`state: disabled`). Native filters out disabled/bypassed blocks using `not block.enabled or block.get_bypassed()`, whereas legacy kept them.

### Connections:
- **Present in Legacy but Missing in Native**:
  - `blocks_char_to_float_0:0->qtgui_time_sink_x_1:0` in `random_bit_generator_dual_sink_sink1_disabled.grc`.
  - **Reason**: The sink block (`qtgui_time_sink_x_1`) is disabled. Native filters out connections where either the source or destination endpoint block is disabled. Legacy kept the connection.

## 3. Parameter Differences & Gaps

Under the strict native filtering rules (`hide == "all"` and `category in {"Advanced", "Config"}`), all parameters present in the legacy API are successfully retained by the native API.

### 3.1 Parameters Added in Native (Missing in Legacy)
Because we purged the ad-hoc regex filter and rely strictly on GRC properties, native retains a richer set of parameters on the `General` tab that legacy failed to capture:
- **`analog_random_source_x`**: `type`
- **`analog_sig_source_x`**: `type`, `phase`, `showports`
- **`blocks_char_to_float`**: `vlen`
- **`blocks_throttle2`**: `type`, `vlen`, `ignoretag`
- **`blocks_add_xx`**: `type`, `vlen`, `num_inputs`
- **`qtgui_time_sink_x`**: `type`, `name`, `ylabel`, `yunit`, `nconnections`, `update_time`, `entags`, `tr_*`, `grid`, `gui_hint`, `ymin`, `ymax`
- **`qtgui_freq_sink_x`**: `average`, `name`, `nconnections`, `norm_window`, `showports`, `tr_chan`, `tr_level`, `tr_mode`, `tr_tag`, `type`, `units`, `update_time`, `wintype`, `autoscale`, `grid`, `gui_hint`, `ymin`, `ymax`, `label`
- **`audio_sink`**: `device_name`, `num_inputs`, `ok_to_block`
- **`qpsk_obj`**: `id`, `npwr`, `soft_dec_lut`

### 3.2 Parameters Dropped in Native (Present in Legacy)
- **None**. Every parameter present in the legacy output is successfully retained by the native output under the strict native filtering pipeline.

All parameter differences are resolved. The native API captures 100% of the legacy parameters, plus additional General-tab/DSP-relevant parameters previously missed by legacy. We can proceed directly to formalizing the target wire shape.
